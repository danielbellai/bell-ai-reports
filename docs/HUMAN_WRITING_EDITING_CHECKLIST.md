# Editing Checklist

Run this checklist against every draft. Go section by section. If anything fails, fix it before delivering the output.

## Punctuation Scan

- [ ] Zero em dashes in the entire document. Search for them. If you find one, replace it with a period, comma, colon, or parenthetical.
- [ ] Zero semicolons in the entire document. If you find one, split it into two sentences.
- [ ] Commas, periods, colons, and parentheses are doing all the punctuation work.

## Vocabulary Scan

- [ ] Zero words from the vocabulary ban list (check verbs, adjectives, nouns, and phrases listed in the main SKILL.md).
- [ ] No more than one instance of any "fancy" word per page. If you used "staggering" on page 2, don't use it again on page 3.
- [ ] "Said" or "says" used for attribution. Not "noted," "emphasized," "highlighted," "observed," "remarked," or "shared."

## Structure Scan

- [ ] No "It's not about X, it's about Y" constructions anywhere.
- [ ] No "From X to Y" range sweeps anywhere.
- [ ] Rule of three used no more than twice in the entire piece.
- [ ] No more than two sentences per page that use the "main clause, -ing phrase" pattern (e.g., "The system reads the message, providing a reply").

## Burstiness Scan

- [ ] At least 3 one-sentence paragraphs in the piece.
- [ ] No sequence of 4 or more sentences with similar length.
- [ ] At least 2 sentences per page that start with "And," "But," "So," or a conversational fragment.

## Stance Scan

- [ ] Every section contains at least one direct opinion from the author.
- [ ] Zero instances of "may potentially," "could arguably," "it's worth noting," or similar hedging filler.
- [ ] The author's voice has personality, not just information.

## Final Read

Read the entire piece one more time, start to finish. Ask yourself: does this sound like a person wrote it, or does it sound like it was assembled? If any sentence makes you pause and think "that's AI," rewrite it.
